# 状态机设计

```mermaid
stateDiagram-v2
    [*] --> INIT_CONTEXT
    INIT_CONTEXT --> INPUT_GUARDRAIL
    INPUT_GUARDRAIL --> ERROR: block
    INPUT_GUARDRAIL --> FINAL: safe_fallback
    INPUT_GUARDRAIL --> RESTORE_MEMORY: allow/mask
    RESTORE_MEMORY --> NORMALIZE_MESSAGES
    NORMALIZE_MESSAGES --> CLASSIFY_INTENT
    CLASSIFY_INTENT --> GENERATE_CLARIFICATION_RESPONSE: low confidence/ambiguous
    GENERATE_CLARIFICATION_RESPONSE --> RESPONSE_PACKAGING
    CLASSIFY_INTENT --> SEMANTIC_RISK_CLASSIFICATION: routed
    SEMANTIC_RISK_CLASSIFICATION --> QUERY_UNDERSTANDING: general
    SEMANTIC_RISK_CLASSIFICATION --> LOAD_BUSINESS_MEMORY: insurance
    QUERY_UNDERSTANDING --> CONTEXT_NEED_PLANNING
    CONTEXT_NEED_PLANNING --> GENERAL_TOOL_ROUTING: tool
    GENERAL_TOOL_ROUTING --> GENERAL_TOOL_CALL: schema valid
    GENERAL_TOOL_CALL --> VERIFY_TOOL_RESULT
    LOAD_BUSINESS_MEMORY --> EXTRACT_INSURANCE_KYC
    EXTRACT_INSURANCE_KYC --> ANALYZE_KYC_AND_ROUTE
    ANALYZE_KYC_AND_ROUTE --> STATUS_ROUTER
    STATUS_ROUTER --> GENERATE_KYC_QUESTIONS: insufficient
    STATUS_ROUTER --> RETRIEVE_INSURANCE_KNOWLEDGE: matched
    STATUS_ROUTER --> BUILD_COMPACT_CONTEXT: unmatched
    RETRIEVE_INSURANCE_KNOWLEDGE --> BUILD_COMPACT_CONTEXT
    BUILD_COMPACT_CONTEXT --> GENERATE_STRATEGY
    GENERATE_KYC_QUESTIONS --> MEMORY_WRITE_PROPOSAL
    GENERATE_STRATEGY --> MEMORY_WRITE_PROPOSAL
    MEMORY_WRITE_PROPOSAL --> VALIDATE_MEMORY_WRITE
    VALIDATE_MEMORY_WRITE --> PERSIST_MEMORY_SNAPSHOT
    PERSIST_MEMORY_SNAPSHOT --> GROUNDING_VERIFICATION
    GROUNDING_VERIFICATION --> OUTPUT_PII_SCAN
    OUTPUT_PII_SCAN --> COMPLIANCE_REVIEW
    COMPLIANCE_REVIEW --> RESPONSE_PACKAGING
    RESPONSE_PACKAGING --> POST_RESPONSE_LOGGER: insurance
    POST_RESPONSE_LOGGER --> SHORT_TERM_MEMORY_UPDATE: sync active intent
    RESPONSE_PACKAGING --> SHORT_TERM_MEMORY_UPDATE: general
    SHORT_TERM_MEMORY_UPDATE --> TRACE_FINALIZE
    TRACE_FINALIZE --> FINAL
    FINAL --> [*]
    ERROR --> [*]
```

向量召回和 LLM 裁定是 `CLASSIFY_INTENT` 内可观测子阶段，通过 Trace 记录 source、TopK 分数、
confidence 和 dispatch action。客户槽位值不进入 Trace。

保险分支刻意在生成追问/策略之后才创建并持久化 `MemoryWriteProposal`。因此只有实际返回给客户的
`presented_kyc_focus` 才会成为已问记录；如果生成失败，不会提前消耗补问轮次。

状态定义在 `graph/state.py`，节点在 `graph/nodes.py`，顺序在 `graph/builder.py`。所有切换必须经过
`move_to()`。
